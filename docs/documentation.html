Koncepcja

Temat

Interpreter języka z możliwością definiowania własnych typów walut.

Semantyka

  • Typowanie statyczne i silne
  • Zmienne mutowalne
  • Do funkcji przekazujemy argumenty przez wartość

Sposób działania

Program interpretera będzie zawierał plik currency.json, który będzie posiadał zdefiniowane waluty i ich kursy. Każda waluta podana w tym pliku będzie nowym typem w języku. Język rzutuje jedną walutę na drugą używając kursu walut.

Przykłady wykorzystania języka

Definiowanie walut

USD := 3.92; /* 4.52 of universial unit */
EUR := 4.53;
PLN := 1;

Porównanie walut

1EUR == 4.52PLN /* true */

Konkatenacja stringów

"aaa" + "bbb" /* "aaabbb" */

Dodawanie z przewalutowaniem

USD a = 1USD;
EUR b = 2EUR;
PLN c = 3PLN;
USD sum = a + b + c; /* 4.16USD */

Procent składany - rekurencja

int power(float basis, int exponent) {
    if (exponent == 0) {
        return 1;
    }
    return basis * power(basis, exponent - 1);
}

USD compound_interest(USD capital, float interest_rate, int number_of_times) {
    return capital * power(1+interest_rate, number_of_times);
}

compound_interest(10USD, 0.05, 5); /* 12.762815USD */

Procent składany - pętla:

USD compound_interest(USD capital, float interest_rate, int number_of_times) {
    int i = number_of_times;
    USD sum = capital;

    while(i == 0) {
        sum = sum * (1 + interest_rate);
        i = i - 1;       
    }
    return sum;
}
compound_interest(10USD, 0.05, 5); /* 12.762815USD */

Porównania z rzutowaniem

int after_how_many_years_dollar_account_rise_to_value_in_pln(USD capital, float interest_rate, PLN value) {
    
    USD sum = capital;
    int counter = 0;

    while(capital > (USD) value) {
        sum = sum * (1 + interest_rate);
        counter = counter + 1;     
    }

    return counter;
}
after_how_many_years_dollar_account_rise_to_value_in_pln(10USD, 0.05, 60PLN) /* 10 */

Obsługa błędów

  • Błąd syntaktyczny - gdy interpreter znajdzie błędny znak lub błędnie umieszczony znak zgłasza błąd wskazując kolumnę i linijkę oraz przerywa działanie
  • Błąd typowania - gdy interpreter znajdzie błędny znak lub błędnie umieszczony typ zgłasza błąd wskazując kolumnę i linijkę oraz przerywa działanie
  • Błąd arytmetyczny (np. dzielenie przez zero) - wskazanie miejsca i przerwanie działania
  • Błąd nazwy - wskazanie miejsca i przerwanie działania

Gramatyka języka

program = declaration, {declaration}, {comment};

declaration = 
   varDeclaration
   | functionDeclaration
;
comment = "/*", {string} "*/";

(* VARIABLE *)
varDeclaration = type, varDeclarationInit;
varDeclarationInit = varDeclarationId, ['=', simpleExpression];
varDeclarationId = ID;

(* FUNCTION *)
functionDeclaration = [type], ID, "(", parms, ")", statement;
parms = {type, ID};

(* STATEMENTS *)
statement = 
   expressionStatement
   | compoundStatement
   | ifStatement
   | whileStatement
   | returnStatement
   | exceptionStatement
;

returnStatement = "return", [expressionStatement], ";";
whileStatement = "while", "(", simpleExpression, ")", statement;
ifStatement = "if", "(" simpleExpression, ")", statement;
compoundStatement = "{", {varDeclaration}, {statement} "}";
expressionStatement = [expression], ";";
exceptionStatement = "try", statement, ["catch", statement];

(* Expressions*)
expression =
   (mutable, "=", expression)
   | simpleExpression
;

simpleExpression = {simpleExpression, "or"}, andExpression;
andExpression = {andExpression, "and"}, reliatonshipExpression;
reliatonshipExpression = sumExpression, [reliatonshipOperand, sumExpression];
sumExpression = sumExpression, sumOperand, multiplyExpression;
multiplyExpression = [multiplyExpression, multiplyOperand], factor;

factor = ["(", type, ")"],
   (
      immutable
      | mutable
   )
;
immutable = 
   "(", expression, ")" 
   | call
   | constant
   ;

mutable = ID;
call = ID, "(", args, ")";
args = {exp};
constant = 
   int
   | float
   | string
   | bool
   | currency
;

reliatonshipOperand = 
   "<="
   | "<"
   | ">",
   | ">=",
   | "==",
   | "!="
;
sumOperand = 
   "+"
   | "-"
;
multiplyOperand =
   "*"
   | "/"
   | "%"
;

type = 
   "int"
   | "float"
   | "string"
   | "bool"
   | CURRENCY_NAME
;

int = 
   [-]
   ,
   "0" 
   | "1-9",{DIGIT}
;

float =
   int, '.' , DIGIT, {DIGIT}
;

currency = 
   (
      int 
      | float
   )
   ,
   CURRENCY_NAME
;

string = "'", {CHAR} "'";

ID = LETTER, {DIGIT | LETTER};
DIGIT = "0-9";
LETTER =
   "a-z"
   | "A-Z"
;
SYMBOL = "[" | "]" | "{" | "}" | "(" | ")" | "<" | ">" | "'" | '"' | "=" | "|" | "." | "," | ";" "_";

CHAR = 
   DIGIT
   | LETTER
   | SYMBOL
;

CURRENCY_NAME = ; (*I.E. 'PLN' | 'EUR' | 'USD';*)

Organizacja projektu

Projekt będzie implementowany w Pythonie.

W funkcji main będą instancje 3 obiektów:

  • Skaner - pobiera dane z wejścia (plik lub string), oraz za pomocą metody get_expression przekazuje dane do znaku końca instrukcji ";"
  • Parser - pobiera dane po instrukcji do instrukcji od skanera. Odbywa się na jego poziomie analiza składniowa i leksykalna. Tokenizuje on dane.
  • Executor - pobiera on dane od Parsera i wykonuje kod programu.

Konwencje

  • currency tylko jako float
  • Id może być pisane tylko snake_casem, bez wielkich liter
  • Nazwa waluty ma 3 znaki z dużej litery
